# Responsible Use Policy

## Purpose

CSharpShellcodeInjector is developed and maintained exclusively for the following purposes:

- **Education** -- understanding low-level Windows internals, PEB/TEB structures, PE export resolution, and managed-to-native interop techniques.
- **Authorized security testing** -- penetration testing and red team engagements performed under explicit written authorization from the system owner.
- **Capture the Flag (CTF)** -- use in sanctioned security competitions and training environments.
- **Security research** -- developing and validating detection signatures, endpoint telemetry, and defensive tooling.

## Prohibited Uses

The following uses of this software are strictly prohibited:

- Accessing, attacking, or testing systems without explicit written authorization from the owner.
- Distributing malware, ransomware, or any payload intended to cause harm to systems or data.
- Embedding this tool in software distributed to end users without full disclosure and consent.
- Using this tool to exfiltrate data, disrupt services, or otherwise violate any applicable law or regulation.
- Circumventing security controls in production environments outside the scope of an authorized engagement.

If you discover a security vulnerability in this project itself (e.g., a flaw in the build process, an unintended information disclosure, or a supply chain concern), please report it responsibly:

1. **Do not** open a public issue containing exploit details.
2. Contact the maintainers privately via the communication channels listed in the repository.
3. Allow a reasonable period for the issue to be addressed before any public disclosure.
